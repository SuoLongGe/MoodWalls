# 本地配置注意事项

本文档说明哪些配置与**个人电脑环境**相关，不应提交到 GitHub；新成员克隆仓库后需按下列步骤完成本地配置。

---

## 一、已被 Git 忽略的文件

以下文件仅存在于你的本机，**不会**被 `git push` 上传：

| 文件 | 用途 |
|------|------|
| `local.properties` | DevEco / HarmonyOS 本地 SDK 路径 |
| `backend/src/main/resources/application-local.yml` | MySQL 密码、JWT 密钥、本机 IP |
| `entry/src/main/ets/common/ApiConfig.local.ets` | 前端连接后端的局域网地址 |

模板文件（**需要提交**，供复制参考）：

| 模板 | 复制为 |
|------|--------|
| `backend/src/main/resources/application-local.yml.example` | `application-local.yml` |
| `entry/src/main/ets/common/ApiConfig.local.example.ets` | `ApiConfig.local.ets` |

## 二、后端配置（Spring Boot）

### 1. 首次配置

```bash
cd backend/src/main/resources
copy application-local.yml.example application-local.yml   # Windows
# cp application-local.yml.example application-local.yml  # macOS / Linux
```

### 2. 编辑 `application-local.yml`

```yaml
spring:
  datasource:
    username: root
    password: 你的MySQL密码

moodwalls:
  jwt:
    secret: 至少32位的随机字符串
  server-host: 你的电脑局域网IP
```

### 3. MySQL 数据库

```bash
mysql -u root -p < backend/sql/init.sql
```

默认库名：`moodwalls`。若端口不是 `3306`，还需修改 `application.yml` 中的 `spring.datasource.url`（该文件为团队共享，一般不用改）。

### 4. 启动后端

```bash
cd backend
mvn spring-boot:run
```

验证：`http://localhost:8080/api/auth/health`

### 5. 端口占用

若提示 `Port 8080 was already in use`：

```powershell
netstat -ano | findstr :8080
taskkill /PID <进程号> /F
```

## 三、前端配置（HarmonyOS）

### 1. 首次配置

```bash
cd entry/src/main/ets/common
copy ApiConfig.local.example.ets ApiConfig.local.ets   # Windows
# cp ApiConfig.local.example.ets ApiConfig.local.ets    # macOS / Linux
```

### 2. 编辑 `ApiConfig.local.ets`

将 `YOUR_LAN_IP` 改为你电脑的**局域网 IPv4 地址**：

```typescript
export function getLocalBaseUrl(): string {
  return 'http://192.168.1.100:8080/api'
}
```

> **重要**：真机或模拟器访问本机后端时，不能使用 `localhost` / `127.0.0.1`，必须使用局域网 IP。

### 3. 查看本机 IP

- Windows：`ipconfig`
- macOS / Linux：`ifconfig` 或 `ip addr`

### 4. 防火墙

确保 Windows 防火墙已放行 **8080** 端口，且手机/模拟器与电脑在同一 Wi‑Fi。

## 四、DevEco Studio 本地路径

`local.properties` 记录 SDK 路径，每人不同，已在 `.gitignore` 中忽略。DevEco 打开项目时会自动生成，无需手动提交。

---

## 五、提交代码前自检

推送前请确认：

- [ ] 未修改 `.gitignore` 去跟踪 `application-local.yml` 或 `ApiConfig.local.ets`
- [ ] `application.yml` 中无真实数据库密码（应使用 `application-local.yml`）
- [ ] `git status` 中不出现上述本地配置文件
- [ ] 未将 `.env`、密钥、AccessKey 等写入仓库

快速检查：

```bash
git status
git diff --cached
```

## 六、团队协作建议

1. **共享配置变更**（如新增 API 路径、端口规范）→ 改 `application.yml` / `ApiConfig.ets` 等已跟踪文件，并更新本文档。
2. **个人环境差异**（密码、IP）→ 只改被忽略的 local 文件。
3. 若后端部署到统一测试服务器，在团队群通知 IP，各成员更新自己的 `ApiConfig.local.ets` 即可。

## 七、常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 前端「网络连接失败」 | IP 错误或后端未启动 | 浏览器访问 `/api/auth/health` 排查 |
| 后端数据库连接失败 | 密码错误或 MySQL 未启动 | 检查 `application-local.yml` |
| 编译报错找不到 `ApiConfig.local` | 未复制 example 文件 | 按第三节步骤创建 |
| 8080 端口冲突 | 旧进程未退出 | 见第二节第 5 点 |
